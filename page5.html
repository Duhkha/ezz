In [1]: import pandas as pd
Data used for this tutorial:
  • Titanic data

    This tutorial uses the Titanic data set, stored as CSV. The data consists of the following data columns:

    • PassengerId: Id of every passenger.

    • Survived: Indication whether passenger survived. 0 for yes and 1 for no.

    • Pclass: One out of the 3 ticket classes: Class 1, Class 2 and Class 3.

    • Name: Name of passenger.

    • Sex: Gender of passenger.

    • Age: Age of passenger in years.

    • SibSp: Number of siblings or spouses aboard.

    • Parch: Number of parents or children aboard.

    • Ticket: Ticket number of passenger.

    • Fare: Indicating the fare.

    • Cabin: Cabin number of passenger.

    • Embarked: Port of embarkation.

How do I read and write tabular data?#

  • I want to analyze the Titanic passenger data, available as a CSV file.

    In [2]: titanic = pd.read_csv("data/titanic.csv")

    pandas provides the read_csv() function to read data stored as a csv file into a pandas DataFrame. pandas supports many different file formats or data sources out of the box (csv, excel, sql, json, parquet, …), each of them with the prefix read_*.

Make sure to always have a check on the data after reading in the data. When displaying a DataFrame, the first and last 5 rows will be shown by default:

    To see the first N rows of a DataFrame, use the head() method with the required number of rows (in this case 8) as argument.

Note

Interested in the last N rows instead? pandas also provides a tail() method. For example, titanic.tail(10) will return the last 10 rows of the DataFrame.

When asking for the dtypes, no brackets are used! dtypes is an attribute of a DataFrame and Series. Attributes of a DataFrame or Series do not need brackets. Attributes represent a characteristic of a DataFrame/Series, whereas methods (which require brackets) do something with the DataFrame/Series as introduced in the first tutorial.

  • My colleague requested the Titanic data as a spreadsheet.

    In [6]: titanic.to_excel("titanic.xlsx", sheet_name="passengers", index=False)

    Whereas read_* functions are used to read data to pandas, the to_* methods are used to store data. The to_excel() method stores the data as an excel file. In the example here, the sheet_name is named passengers instead of the default Sheet1. By setting index=False the row index labels are not saved in the spreadsheet.

The equivalent read function read_excel() will reload the data to a DataFrame:

In [7]: titanic = pd.read_excel("titanic.xlsx", sheet_name="passengers")

    The method info() provides technical information about a DataFrame, so let’s explain the output in more detail:

    • It is indeed a DataFrame.

    • There are 891 entries, i.e. 891 rows.

    • Each row has a row label (aka the index) with values ranging from 0 to 890.

    • The table has 12 columns. Most columns have a value for each of the rows (all 891 values are non-null). Some columns do have missing values and less than 891 non-null values.

    • The columns Name, Sex, Cabin and Embarked consists of textual data (strings, aka object). The other columns are numerical data with some of them whole numbers (aka integer) and others are real numbers (aka float).

    • The kind of data (characters, integers,…) in the different columns are summarized by listing the dtypes.

    • The approximate amount of RAM used to hold the DataFrame is provided as well.

REMEMBER

  • Getting data in to pandas from many different file formats or data sources is supported by read_* functions.

  • Exporting data out of pandas is provided by different to_*methods.

  • The head/tail/info methods and the dtypes attribute are convenient for a first check.

Example 4: Sales Data with Regional Information Analysis

(https://pandas.pydata.org/pandas-docs/stable/getting_started/intro_tutorials/02_read_write.html) In this example, we will:

  1. Load the Sales Data from a CSV file.
    • Address any delimiter or encoding issues.
    • Convert the Date column to datetime and set it as the index.
  2. Data Cleaning & Preprocessing:
    • Handle missing values and duplicate records.
    • Ensure that columns have the appropriate data types.
  3. Exploratory Data Analysis (EDA):
    • Generate descriptive statistics.
    • Visualize overall sales trends over time.
    • Group data by Region to compare average (or total) sales.
    • Create relevant visualizations (line charts, bar plots, etc.).
Note: Ensure you have a file called sales_data.csv in your working directory. 

  import pandas as pd
  import numpy as np
  import matplotlib.pyplot as plt
  import seaborn as sns
  
  sns.set_style("whitegrid")

1. Data Loading & Initial Inspection

File Parameters:
- File Name: sales_data.csv
- Expected columns: Region, Sales, Date, Product.
- The file is assumed to be comma-delimited.

We'll load the CSV file, inspect the first few rows, get the shape, and review the data types. 

  # Define file parameters
  file_path = 'sales_data.csv'
  csv_delimiter = ','       # Change if your file uses a different delimiter (e.g., '\t')
  encoding_used = 'utf-8'   # Adjust encoding if necessary
  
  # Load the CSV file
  try:
      df = pd.read_csv(file_path, delimiter=csv_delimiter, encoding=encoding_used)
      print("CSV file loaded successfully!")
  except Exception as e:
      print("Error loading file:", e)

Initial Inspection:
- View the first five rows.
- Check dataset shape and data types.
- Display summary statistics. 

  print("First 5 rows:")
  display(df.head())
  
  print("Dataset shape (rows, columns):", df.shape)
  
  print("\nDataset Info:")
  df.info()
  
  print("\nSummary Statistics:")
  display(df.describe(include='all'))

2. Data Cleaning & Preprocessing

Key Cleaning Tasks:
- Convert the Date column to datetime and set it as the index.
- Handle missing values (e.g., fill numeric sales data with the mean or drop rows).
- Remove any duplicate rows.
- Ensure regional information (and other categorical columns) is formatted properly. 

  # Convert the 'Date' column to datetime
  if 'Date' in df.columns:
      try:
          df['Date'] = pd.to_datetime(df['Date'], errors='coerce')
          print("Converted 'Date' to datetime.")
      except Exception as e:
          print("Error converting 'Date':", e)
  else:
      print("'Date' column not found.")
  
  # Set 'Date' as the index for time series analysis
  if 'Date' in df.columns:
      df.set_index('Date', inplace=True)
      print("Set 'Date' as the index.")
      
  # Check for missing values before cleaning
  print("Missing values per column before cleaning:")
  print(df.isna().sum())
  
  # Create a copy for cleaning
  df_clean = df.copy()
  
  # Identify numeric columns (e.g., 'Sales') and categorical columns (e.g., 'Region', 'Product')
  numeric_cols = df_clean.select_dtypes(include=['number']).columns
  categorical_cols = df_clean.select_dtypes(include=['object']).columns
  
  # Impute numeric columns with their mean (if suitable)
  for col in numeric_cols:
      df_clean[col].fillna(df_clean[col].mean(), inplace=True)
  
  # Fill categorical columns with a placeholder ('Unknown')
  for col in categorical_cols:
      df_clean[col].fillna('Unknown', inplace=True)
  
  print("Missing values per column after cleaning:")
  print(df_clean.isna().sum())
  
  # Remove duplicate rows
  duplicates_count = df_clean.duplicated().sum()
  print("Number of duplicate rows before removal:", duplicates_count)
  df_clean = df_clean.drop_duplicates()
  print("Shape after removing duplicates:", df_clean.shape)

3. Exploratory Data Analysis (EDA)

We'll now analyze the cleaned sales data to uncover trends and patterns.

3.1 Descriptive Statistics

Display overall statistics for the numeric variables (e.g., Sales) and a full summary. 

  print("Descriptive Statistics for Numeric Variables:")
  display(df_clean.describe())
  
  print("Overall Descriptive Statistics:")
  display(df_clean.describe(include='all'))

3.2 Sales Trends over Time

- Plot overall sales as a time series to observe trends.
- You can resample the data by day, week, or month for clearer trends. 

  if 'Sales' in df_clean.columns:
      # Resample by month and sum the sales
      monthly_sales = df_clean['Sales'].resample('M').sum()
      plt.figure(figsize=(10, 5))
      plt.plot(monthly_sales.index, monthly_sales, marker='o')
      plt.title('Total Monthly Sales')
      plt.xlabel('Date')
      plt.ylabel('Total Sales')
      plt.xticks(rotation=45)
      plt.show()
  else:
      print("'Sales' column not found.")

3.3 Grouping and Aggregation by Region

- Group data by the Region column.
- Compute the average or total sales for each region.
- Visualize the results using a bar chart. 

  if 'Region' in df_clean.columns and 'Sales' in df_clean.columns:
      region_sales = df_clean.groupby('Region')['Sales'].sum().reset_index()
      print("Total Sales by Region:")
      display(region_sales)
      
      plt.figure(figsize=(8, 4))
      sns.barplot(data=region_sales, x='Region', y='Sales')
      plt.title('Total Sales by Region')
      plt.xlabel('Region')
      plt.ylabel('Total Sales')
      plt.xticks(rotation=45)
      plt.show()
  else:
      print("Either 'Region' or 'Sales' column is missing.")

3.4 Additional Visualizations (Optional)

Optional ideas:

  • Sales Distribution: Create a histogram of the sales figures.
  • Product Analysis: If the Product column is available, group by product to analyze sales per product category.

  # Sales Distribution: Histogram
  if 'Sales' in df_clean.columns:
      plt.figure(figsize=(8, 4))
      sns.histplot(df_clean['Sales'], kde=True, bins=30)
      plt.title('Distribution of Sales')
      plt.xlabel('Sales')
      plt.ylabel('Frequency')
      plt.show()
  
  # Optional: Group by Product (if exists)
  if 'Product' in df_clean.columns and 'Sales' in df_clean.columns:
      product_sales = df_clean.groupby('Product')['Sales'].sum().reset_index()
      print("Total Sales by Product:")
      display(product_sales)
      
      plt.figure(figsize=(8, 4))
      sns.barplot(data=product_sales, x='Product', y='Sales')
      plt.title('Total Sales by Product')
      plt.xlabel('Product')
      plt.ylabel('Total Sales')
      plt.xticks(rotation=45)
      plt.show()
  else:
      print("Optional Product analysis skipped: 'Product' or 'Sales' column missing.")

Recap of Example 4

What We Did:

  1. Data Loading & Inspection:
    Loaded sales_data.csv while addressing delimiter/encoding issues, inspected the data (first few rows, shape, data types, and summary statistics).
  2. Data Cleaning & Preprocessing:
    Converted the Date column to datetime and set it as the index; handled missing values (numeric imputation and categorical substitution) and removed duplicates.
  3. Exploratory Data Analysis (EDA):
    Generated descriptive statistics, visualized sales trends over time by resampling the data, grouped data by Region to compare total sales, and optionally examined sales distributions and product-level details.
This workflow demonstrates how to approach a dataset with regional sales information, covering a variety of essential data analysis tasks.
