Step 1: Data Loading & Initial Inspection

https://pandas.pydata.org/pandas-docs/stable/getting_started/intro_tutorials/04_plotting.html# In this step, you will:

  1. Load the Dataset:
    • Work with either CSV or JSON files.
    • Handle potential issues like different delimiters (e.g., commas, tabs), and encoding settings.
  2. Initial Inspection:
    • Check the first few rows of data.
    • Understand data dimensions (rows and columns).
    • Review data types and basic summary statistics.
  3. Detect and Handle Date/Time Columns (if applicable):
    • Identify any columns that might represent dates or times.
    • Convert them properly (using pd.to_datetime()) and, if needed, set as the DataFrame index.

Note: This step is broad since many issues (like wrong delimiters or encoding) might arise when loading data.

import pandas as pd
import numpy as np

1.1. File Path, Format, and Optional Parameters

Set up your file path and indicate the file format (csv or json).

Tips:
CSV Files:

  • May use different delimiters (e.g., comma ,, tab \t, or semicolon ;).
  • Encoding issues may occur; you might need to specify encoding='utf-8' or another encoding.

JSON Files:
Typically straightforward with pd.read_json(), but the structure may vary (nested objects, lists, etc.).

Customize these parameters based on the dataset you receive. 

# Define file parameters
file_path = 'dataset.csv'      # Change to 'dataset.json' as needed
file_format = 'csv'            # Set 'csv' or 'json'

# For CSV files, you may need to specify a delimiter if it is not a comma, e.g.:
csv_delimiter = ','            # Change to '\t', ';', etc., if necessary
encoding_used = 'utf-8'        # Adjust encoding if there is an error loading the data

1.2. Load the Dataset

Use the appropriate loading function and adjust based on the file parameters.

Common Adjustments:

  • Delimiter Issues (CSV): Pass the delimiter=csv_delimiter parameter.
  • Encoding: Specify encoding=encoding_used if needed.
  • Errors: Include error handling to catch and report any issues while loading.
try:
    if file_format.lower() == 'csv':
        # For CSV files, you might encounter non-default delimiters or encodings.
        df = pd.read_csv(file_path, delimiter=csv_delimiter, encoding=encoding_used)
        print("CSV file loaded successfully!")
    elif file_format.lower() == 'json':
        df = pd.read_json(file_path)
        print("JSON file loaded successfully!")
    else:
        raise ValueError("Unsupported file format. Please specify either 'csv' or 'json'.")
except Exception as e:
    print("Error loading file:", e)

1.3. Initial Data Inspection

Once the data is loaded, perform common inspection tasks:

  • View the Data: Use .head() to see the first few rows.
  • Data Dimensions: Use .shape to check the number of rows and columns.
  • Data Types and Null Values: Use .info() to get details about data types and missing values.
  • Summary Statistics: Use .describe(include='all') to view a statistical summary.
# Display the first few rows of the dataset
print("First 5 rows:")
display(df.head())

# Check dataset dimensions
print("Dataset shape (rows, columns):", df.shape)

# Print detailed information about data types and missing values
print("\nDataset Info:")
df.info()

# Get summary statistics for all columns
print("\nSummary Statistics:")
display(df.describe(include='all'))

1.4. Special Considerations for Date/Time Columns (Time Series Data)

For time series datasets, you'll typically have a date or time column.

What to do:

  • Look for a column with a name that suggests it is a date/time column (e.g., "Date", "Timestamp", "Time").
  • Convert this column using pd.to_datetime().
  • Optionally, set it as the DataFrame index to facilitate time-based operations later.
# Identify potential date or time columns by searching for 'date' or 'time' in the column names
date_columns = [col for col in df.columns if 'date' in col.lower() or 'time' in col.lower()]

if date_columns:
    print("\nPotential date/time column(s) detected:", date_columns)
    # Assume the first matched column is our date column for simplicity
    date_col = date_columns[0]
    try:
        df[date_col] = pd.to_datetime(df[date_col], errors='coerce')
        print(f"Column '{date_col}' successfully converted to datetime.")
        # Optionally, set the date column as the index for time series analysis
        df.set_index(date_col, inplace=True)
        print(f"Column '{date_col}' is now set as the DataFrame index.")
    except Exception as e:
        print(f"Error converting column '{date_col}' to datetime:", e)
else:
    print("\nNo obvious date/time column found. Assuming traditional tabular data.")

1.5. Recap and Potential Issues

Key points to remember:

  • File Format Differences:
    • CSV: May require adjustments for delimiters (delimiter=) and encodings (encoding=).
    • JSON: Load directly with pd.read_json(), but be ready for structural differences.
  • Date/Time Handling:
    • Critical in time series data; always check for date/time columns.
    • Use pd.to_datetime() to ensure proper formatting and consider setting as index.
  • Initial Inspections: Always check the first rows, shape, data types, and summary statistics.

This concludes the broad outline for Step 1. Next steps (data cleaning, transformation, analysis) will build upon this foundation.

Step 2: Data Cleaning & Preprocessing

In this step, we will prepare the data for analysis by addressing common issues.

Objectives:

  1. Handling Missing Values:
    • Identify missing data.
    • Decide whether to drop or impute missing values.
  2. Handling Duplicates:
    • Identify and remove duplicate rows.
  3. Data Type Corrections:
    • Ensure that each column has the appropriate data type.
  4. Special Considerations for Time Series Data:
    • If you have a date/time index, ensure that it’s properly sorted.
    • Create any additional features if needed (e.g., extracting month, day, etc.).

Note: Focus on solving delimiter issues, encoding problems, or date parsing issues have been addressed in Step 1.

2.1. Handling Missing Values

Why? Missing values can distort the analysis and reduce the quality of insights.
Common Strategies:

  • Drop Rows/Columns: Remove rows or columns where data is insufficient.
  • Imputation: Replace missing values with a statistic (mean, median, or mode) or forward/backward fill for time series.

Below are examples for both approaches.

# 2.1.a. Identify Missing Values
print("Missing values per column before cleaning:")
print(df.isna().sum())

Option 1: Dropping Missing Values

You might choose to drop rows or columns where missing data is found.

# Option: Drop rows that contain any missing values
df_drop = df.dropna()
print("\nShape after dropping rows with any missing values:", df_drop.shape)

# If appropriate, you could drop columns instead:
# df_drop_cols = df.dropna(axis=1)
# print("Shape after dropping columns with any missing values:", df_drop_cols.shape)

Option 2: Imputing Missing Values

For numerical columns, a common strategy is to fill missing values with the mean or median. For categorical columns, you might fill with the most frequent value or a placeholder. 

# Create a copy to fill missing values
df_impute = df.copy()

# Identify numeric and categorical columns
numeric_cols = df_impute.select_dtypes(include=['number']).columns
categorical_cols = df_impute.select_dtypes(exclude=['number', 'datetime']).columns

# Impute numeric columns with the mean (or median as needed)
for col in numeric_cols:
    df_impute[col].fillna(df_impute[col].mean(), inplace=True)
    
# Impute categorical columns with a placeholder value, e.g., 'Unknown'
for col in categorical_cols:
    df_impute[col].fillna('Unknown', inplace=True)

print("Missing values per column after imputation:")
print(df_impute.isna().sum())

2.2. Handling Duplicates

Why? Duplicates can skew analysis and lead to biased results.
Action: Remove duplicate rows from the dataset. 

# Check for duplicates
duplicates_count = df_impute.duplicated().sum()
print("Number of duplicate rows before cleaning:", duplicates_count)

# Remove duplicates
df_clean = df_impute.drop_duplicates()

print("Shape after removing duplicates:", df_clean.shape)

Common Tasks:

  • Convert strings representing numbers to numeric types.
  • Verify date/time columns (already addressed, but check for remaining issues).
  • Adjust categorical variables if needed.
# Example: Convert a column (if necessary) to numeric type using pd.to_numeric.
# Uncomment and adjust the column name as needed.

# Re-inspect the information after cleaning

df_clean.info()

2.4. Special Considerations for Time Series Data

If your dataset is a time series (i.e., has a datetime index), then consider the following:

  • Sorting: Ensure that the DataFrame is sorted by the date/time index.
  • Feature Engineering: You might extract additional features such as year, month, day, weekday, etc.
if isinstance(df_clean.index, pd.DatetimeIndex):
    # Sort by the date/time index
    df_clean.sort_index(inplace=True)
    print("Data sorted by date/time index.")

    # Optionally, create new features from the datetime index:
    df_clean['Year'] = df_clean.index.year
    df_clean['Month'] = df_clean.index.month
    df_clean['Day'] = df_clean.index.day
    df_clean['Weekday'] = df_clean.index.dayofweek
    print("Extracted additional time features:")
    display(df_clean.head())
else:
    print("No datetime index detected; skipping time series specific cleaning.")

Step 3: Exploratory Data Analysis (EDA)

In this step, we explore the cleaned dataset to uncover patterns, trends, and basic relationships. The objectives are to:

  1. Generate Descriptive Statistics:
    • Examine central tendencies (mean, median) and the spread (standard deviation, range) for numeric features.
    • Review frequency distributions for categorical features.
  2. Perform Grouping and Aggregation:
    • Use group operations to compare subsets (if there are categorical variables).
  3. Visualize Data:
    • Plot distributions using histograms, box plots, or density plots.
    • Visualize relationships using scatter plots, correlation heatmaps, or bar charts.
    • For time series data, plot trends over time (line plots) if the dataset has a date/time index.

This exploratory analysis provides the foundation for deeper investigation in subsequent steps.

3.1. Descriptive Statistics

Overall Statistics:
Use .describe() to review key metrics for numeric features.
Include all columns to see stats for categorical features, too. 

# Display overall descriptive statistics for numeric columns
print("Descriptive Statistics for Numeric Variables:")
display(df_clean.describe())

# If you want to see summary stats for all columns (including categorical)
print("Descriptive Statistics for All Variables:")
display(df_clean.describe(include='all'))

3.2. Analyzing Data Distributions

Distribution for Numeric Variables:
Histograms or density plots help you see how numeric data is distributed.
Box plots are useful to detect outliers.

Example: Create a histogram for one numeric column. 

import matplotlib.pyplot as plt
import seaborn as sns

# Identify a numeric column to explore (if available)
numeric_columns = df_clean.select_dtypes(include=['number']).columns
if len(numeric_columns) > 0:
    col_to_plot = numeric_columns[0]  # change index if needed
    plt.figure(figsize=(8, 4))
    sns.histplot(df_clean[col_to_plot], kde=True, bins=30)
    plt.title(f'Histogram and KDE of {col_to_plot}')
    plt.xlabel(col_to_plot)
    plt.ylabel('Frequency')
    plt.show()

    # Additionally, display a box plot to detect outliers
    plt.figure(figsize=(6, 3))
    sns.boxplot(x=df_clean[col_to_plot])
    plt.title(f'Box Plot of {col_to_plot}')
    plt.show()
else:
    print("No numeric columns available for distribution plots.")

3.3. Grouping and Aggregation

If your dataset contains categorical variables (e.g., 'Category', 'Type'):
Perform grouping operations to compute means, counts, or other aggregate statistics.
Example: Group the data by a categorical variable and calculate the average of a numeric feature. 

# Identify a categorical column if available for grouping
categorical_columns = df_clean.select_dtypes(include=['object']).columns

if len(categorical_columns) > 0 and len(numeric_columns) > 0:
    group_col = categorical_columns[0]  # change as needed
    numeric_col = numeric_columns[0]    # change as needed
    grouped = df_clean.groupby(group_col)[numeric_col].mean().reset_index()

    print(f"Average of {numeric_col} grouped by {group_col}:")
    display(grouped)

    # Visualize the grouped data using a bar chart
    plt.figure(figsize=(8, 4))
    sns.barplot(data=grouped, x=group_col, y=numeric_col)
    plt.title(f'Average {numeric_col} by {group_col}')
    plt.xlabel(group_col)
    plt.ylabel(f'Average {numeric_col}')
    plt.xticks(rotation=45)
    plt.show()
else:
    print("No appropriate categorical or numeric columns found for grouping analysis.")

3.4. Correlation Analysis

Purpose: Identify relationships between numeric features.
Example: Compute and plot a correlation heatmap. 

if len(numeric_columns) >= 2:

Each of the plot objects created by pandas is a Matplotlib object. As Matplotlib provides plenty of options to customize plots, making the link between pandas and Matplotlib explicit enables all the power of Matplotlib to the plot. This strategy is applied in the previous example:

fig, axs = plt.subplots(figsize=(12, 4))        # Create an empty Matplotlib Figure and Axes
air_quality.plot.area(ax=axs)                   # Use pandas to put the area plot on the prepared Figure/Axes
axs.set_ylabel("NO$_2$ concentration")          # Do any Matplotlib customization you like
fig.savefig("no2_concentrations.png")           # Save the Figure/Axes using the existing Matplotlib method.
plt.show()                                      # Display the plot

REMEMBER

  • The .plot.* methods are applicable on both Series and DataFrames.

  • By default, each of the columns is plotted as a different element (line, boxplot,…).

  • Any plot created by pandas is a Matplotlib object.
